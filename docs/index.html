Engineering Signals

API-first delivery

OpenAPI is the source of truth. Spring MVC interfaces and DTOs are generated from the contract.

Clear API evolution

Version 1 keeps the minimal stable user contract. Version 2 adds profile fields and partial updates.

Production habits

BCrypt password hashing, soft deletes, PostgreSQL migrations and centralized problem responses.

Reviewable quality

Mockito web/service tests, Testcontainers integration wiring and GitHub Actions CI.

HTTP surface

Versioned User API

Method Path Intent
POST /api/v1/users Create a minimal v1 user
GET /api/v1/users/{id} Read the v1 representation
DELETE /api/v1/users/{id} Soft-delete through inactive status
POST /api/v2/users Create a v2 user with profile fields
GET /api/v2/users/{id} Read the v2 representation
PATCH /api/v2/users/{id} Partially update profile or status
